## 核心设计原则

### 1. 客户端验证架构

- **服务器权威**：实体位置和碰撞以服务器数据为准
- **客户端同步**：客户端从服务器获取实体数据进行渲染
- **边界验证**：客户端实时拦截无效移动，减少服务器压力

### 2. 地形碰撞规则

#### 水系（河流 vs 桥梁）

```
河流本身（河面）→ 不可通行
桥梁/道路覆盖区域（在河流坐标范围内）→ 允许通行
```

**实现逻辑**：
1. 先检查点是否在河流范围内（点到折线距离 < 宽度/2）
2. 如果在河流内 → 再检查是否在桥梁范围内
3. 在桥/路上 → 允许通行
4. 不在桥/路上 → 阻止移动，提示"请走桥梁"

#### 湖泊

- 湖泊区域（椭圆区域）→ 不可通行
- 湖泊边界（边缘）→ 不可通行

#### 海洋

- 海洋（z > 90）→ 不可通行
- 海滩（z = 85~90）→ 允许通行（不是水系）

### 3. 坐标系统

- 世界范围：x ∈ [-100, 100], z ∈ [-100, 100]
- 河流宽度：8 单位
- 湖泊：椭圆中心 (10, -75)，半径 55×22
- 海洋边界：z = 90

### 4. 实体边界

- **建筑**：碰撞箱基于贴近地面的 footprint，而非顶部轮廓
- **变形金刚**：机器人形态脚底对齐地面
- **坦克**：同样脚底对齐地面

### 5. 音效系统

- **引擎音效**：连接到 `AmbientSoundSystem.masterGain`，受系统音量控制
- **背景音乐**：由 `AmbientSoundSystem` 管理，与环境音并行播放
- **AudioContext 复用**：所有音效系统必须复用同一个 AudioContext
