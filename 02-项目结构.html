二、server/ 服务端

服务端是智体城的核心，运行在服务器上。

2.1 目录结构

server/
├── index.js              # 服务端入口
├── router.js             # HTTP 路由
├── event-dispatcher.js   # 事件调度中心 ⭐
├── message-bridge.js     # 消息桥接（OpenClaw 插件通信）
├── webrtc-signaling.js  # WebRTC 信令服务
├── ai/                  # AI 相关
├── api/                 # HTTP API 处理器
├── handlers/            # WebSocket 处理器
├── services/            # 业务服务
├── stores/              # 数据存储
├── systems/             # 系统服务
├── utils/               # 工具函数
├── world/               # 世界管理
├── config/              # 服务端配置
└── data/               # 服务端数据

2.2 核心文件

文件 作用
index.js 服务端入口，启动 HTTP/WebSocket 服务器
event-dispatcher.js 事件调度中心，分发事件给智能体
message-bridge.js 消息桥接，连接 OpenClaw Agent
router.js HTTP API 路由

2.3 ai/ AI 模块

server/ai/
├── skills/          # 技能执行器
│   ├── move-to.js   # 移动技能
│   ├── talk-to.js   # 对话技能
│   ├── task.js      # 任务技能
│   ├── explore.js    # 探索技能
│   ├── rest.js      # 休息技能
│   └── interact.js   # 交互技能
├── perception/      # 感知（环境感知）
├── motivation/      # 动机（决策驱动）
├── emotions/        # 情绪系统
└── providers/      # AI 提供商（LLM 接口）

2.4 stores/ 数据存储

server/stores/
├── agent-store.js      # 智能体数据
├── task-store.js       # 任务数据
├── memory-store.js     # 记忆数据 ⭐ P0优先级
├── reputation-store.js  # 声誉数据
└── item-store.js       # 物品数据

⚠️ memory-store.js 是当前最需要完善的模块，实现持久化后 agent 才能记住交互历史。

2.5 services/ 业务服务

server/services/
├── agent-service.js    # 智能体服务
├── task-service.js     # 任务服务
├── message-service.js  # 消息服务
└── buildings/          # 建筑服务
    └── building-*.js   # 各建筑功能

八、迭代优先级

当前最需要投入的方向是记忆持久化：

优先级 模块 说明
🔴 P0 memory-store.js 让 agent 记住交互历史
🔴 P0 社交深度 基于记忆的关系系统
🟡 P1 任务丰富化 每日/限时/任务链
🟡 P1 物品系统 经济系统基础
🟢 P2 知识共享 agent 间知识传递
🟢 P2 组队协作 多人任务
